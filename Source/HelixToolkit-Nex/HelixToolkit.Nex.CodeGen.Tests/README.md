# ObservablePropertyGenerator Tests

This project contains comprehensive tests for the `ObservablePropertyGenerator` source generator.

## Test Structure

### ObservablePropertyGeneratorTests
Uses the Roslyn Source Generator Testing framework to verify the generator produces correct output.

**Test Cases:**
- `GeneratesBackingFieldAndProperty_WithDefaultValue` - Verifies generation with default values
- `GeneratesBackingFieldAndProperty_WithoutDefaultValue` - Verifies generation without defaults
- `GeneratesMultipleProperties_InSingleClass` - Tests multiple properties in one class
- `DoesNotGenerate_ForNonPartialClass` - Ensures non-partial classes are skipped
- `DoesNotGenerate_ForReadOnlyProperty` - Ensures read-only properties are skipped
- `GeneratesCorrectCamelCaseFieldName` - Verifies camelCase field naming
- `GeneratesWithComplexDefaultValue` - Tests complex initialization expressions

### ObservablePropertyGeneratorIntegrationTests
Integration tests that compile code with the generator and verify runtime behavior.

**Test Cases:**
- `Generator_ProducesNoErrors_WithValidInput` - Ensures no compilation errors
- `Generator_ProducesExpectedNumberOfFiles` - Verifies correct file count
- `Generator_GeneratesCorrectFileName` - Checks generated file naming
- `Generator_GeneratesValidCSharpCode` - Validates generated code syntax
- `Generator_HandlesMultiplePropertiesInClass` - Tests multiple property generation
- `Generator_IgnoresPropertiesWithoutObservableAttribute` - Verifies attribute filtering
- `Generator_PreservesNamespace` - Ensures namespace is preserved
- `Generator_GeneratesAutoGeneratedComment` - Checks for auto-generated markers

## Running the Tests

### Visual Studio
1. Open Test Explorer (Test > Test Explorer)
2. Click "Run All" or run individual tests

### Command Line
```bash
dotnet test
```

### With Coverage
```bash
dotnet test /p:CollectCoverage=true
```

## Expected Test Behavior

All tests should pass when:
1. The `ObservablePropertyGenerator` correctly implements `IIncrementalGenerator`
2. The `ObservableAttribute` is available in the `HelixToolkit.Nex` namespace
3. The `ObservableObject` base class is available with the `Set` method

## Troubleshooting

### Tests Fail to Find ObservableObject
Ensure the `HelixToolkit.Nex` project is built before running tests.

### Generator Not Producing Output
Check that:
- Classes are marked as `partial`
- Properties have both getter and setter
- The `[Observable]` attribute is applied correctly

### Compilation Errors in Tests
Verify all required assembly references are included in the test project.

## Example Usage

The generator transforms this:

```csharp
public partial class MyMaterial : ObservableObject
{
    [Observable(Default = "Vector4.One")]
    public Vector4 Color { get; set; }
}
```

Into this:

```csharp
public partial class MyMaterial
{
    private Vector4 _color = Vector4.One;
    
    public Vector4 Color
    {
        get => _color;
        set { Set(ref _color, value); }
    }
}
```
